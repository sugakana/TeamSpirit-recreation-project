# API詳細設計書　プロンプト

## 作成例
```
@api_detailed_design_rules.mdc を参考に
@feature-list.md のバックエンドのUWPにある商品検索処理の詳細背経書を作成して
利用するDBはDyanmoDBで
@entities がエンティティ一覧である。
本機能の目的は、レジに表示するための商品の情報を取得することが目的であり、ユーザーによって制限されている商品がある。制限されている商品は、表示フラグをfalseにする。また、カテゴリを選択するたびに全ての商品を取得するのではなく、全ての商品情報を取得する。
更新日：[本日の日付]
更新者：[自身のアカウント名]
```
